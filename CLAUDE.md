## Структура данных

## Правила и почему они такими стали

### Интерфейс
- Крупные кнопки min 44px — приложение для телефона, 
  пользователь часто держит его одной рукой
- Адаптивность под мобильный обязательна — десктоп вторичен
- Максимум информации на карточке: название, место, количество, 
  срок — больше не помещается на экране телефона без скролла

### Данные и даты
- Для работы с датами использовать dayjs — не использовать 
  date() напрямую, были баги с часовыми поясами
- Срок годности хранить в формате YYYY-MM-DD
- id генерировать через Date.now() — достаточно для одного 
  пользователя

### Цвета и стиль
- Светлый фон, зелёные и бежевые акценты — аптечная эстетика, 
  вызывает доверие
- Карточки: белый = норма, жёлтый = истекает через N дней, 
  красный = просрочено
- Порог по умолчанию 7 дней — настраивается пользователем 
  в настройках
